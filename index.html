1. ConnectionFactory (Pooling)

Penjelasan: Digunakan untuk membatasi jumlah koneksi. Objek disimpan dalam sebuah pool (list) agar bisa digunakan berulang kali daripada membuat koneksi baru setiap saat. 
static final List _pool = ['Conn-A', 'Conn-B'];

2. NotificationFactory (Polymorphism)

Penjelasan: Membungkus pembuatan objek. Kita tidak memanggil new EmailNotification(), tapi cukup meminta ke factory. Ini membuat kode utama lebih rapi. 
static Notification create(String type) => type == 'email' ? EmailNotification() : PushNotification();

3. ShapeFactory (Caching)

Penjelasan: Menggunakan Map sebagai cache. Jika objek yang sama diminta kembali, ia akan mengambil dari memory (cache) alih-alih membuat objek baru yang memakan memori. 
static final Map _cache = {};
